# Smith - 4分ハッカソンデモ動画構成

## Context

ハッカソンプロダクト「Smith」の4分プレゼン動画。Speculative Turn-taking（意味ベースの割り込み検出）が核心技術。審査員に「これは既存のボイスAIと根本的に違う」と思わせることが目標。

---

## 0:00-0:20 — フック（問題提示）

**画面**: 既存のボイスAI（Siri/Alexa等）で「うんうん」と相槌を打つと会話が止まる映像
**ナレーション**:
> 「ボイスAIと話していて、相槌を打っただけで会話が止まった経験、ありませんか？」
> 「音量ベースのVADは、意味を理解しない。Smithはこの問題を解決します。」

---

## 0:20-1:00 — コア技術: Speculative Turn-taking（40秒）

**画面**: アーキテクチャ図 → ライブデモ（DivergenceIndicatorを画面に表示）

**3フェーズを視覚的に説明**:

1. **予測フェーズ**: AIが話している間にGemini Flashが5つの応答を予測（画面に予測リスト表示）
2. **リアルタイム評価**: ユーザーの発話をembedding → cosine距離で比較（10ms以下）
3. **判定**:
   - スコア < 0.3 → 相槌 → **AIは話し続ける**（デモ: 「うんうん」で止まらない）
   - スコア > 0.6 → 本物の反論 → **AIが即座に譲る**（デモ: 「いや、それは違う」で即停止）

**ライブ比較デモ**:

- Speculative OFF: 「うんうん」→ AIが止まる（悪い体験）
- Speculative ON: 「うんうん」→ AIは止まらない。「それは違う」→ 即停止（良い体験）

## 1:00-2:20 — プロダクトデモ: 要件定義ミーティング（80秒）

**シナリオ**: ECサイトの要件定義をSmithと行う

### 1:00-1:20 — ミーティング開始

**画面**: ログイン → MeetingRoom画面 → Simliアバターが動く
**操作**: マイクON → 「ECサイトを作りたい。商品検索と決済機能が必要」
**見せるポイント**:

- Simliリップシンクアバターがリアルタイムで動く
- トランスクリプトがリアルタイム表示

### 1:20-1:50 — 4ペインダッシュボード自動生成

**画面**: 4ペインが次々と自動的に埋まっていく様子
**見せるポイント**:

- **Outline**: 要件ツリーが自動構築（requirement/goal/assumption）
- **Graffle**: アーキテクチャ図が自動生成（フロント→API→DB）
- **Focus**: タスクがKanbanに自動追加（todo/in_progress/done）
- **Plan**: Ganttチャートにスケジュールが自動配置

**ナレーション**: 「会話しているだけで、要件・設計・タスク・スケジュールが自動生成されます」

### 1:50-2:20 — 双方向編集

**操作**: Outlineのノードを手動編集 → AIがその変更を認識して会話に反映
**見せるポイント**:

- ユーザーがダッシュボードを編集 → ADKエージェントがuser_focusを検知
- AIが「今編集された部分について補足すると...」と文脈を理解
- 画面共有機能でモックアップを見せる → AIがデザインにコメント

## 2:20-3:00 — 技術アーキテクチャ（40秒）

**画面**: アーキテクチャ概要図

```text
Browser (Next.js + Firebase Auth)
    ↕ WebSocket (PCM16 audio + JSON)
Cloud Run (FastAPI)
    ├── Gemini Live API (音声対話)
    ├── Speculative Engine (予測 + 埋め込み比較)
    ├── ADK Background Agent (15秒毎にダッシュボード更新)
    ├── Simli Avatar (リップシンク映像)
    └── Firestore (リアルタイム同期)
```

**キーポイント**:

- マルチエージェント: Gemini Live（対話） + ADK Agent（構造化） + Speculative Engine（割り込み判定）が並行動作
- Firestoreリアルタイム同期で遅延ゼロのUI更新
- 全てGCP上で動作（Cloud Run, Firestore, Gemini API）
